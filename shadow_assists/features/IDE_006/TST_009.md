# TST_009 — Reload prefetch 동작 검증

| 항목 | 값 |
|---|---|
| ID | `TST_009` |
| 상태 | `재정의 (TSK_012 의 일부로 통합)` (2026-04-28) |
| 검증 대상 | **[`TSK_009`](TSK_009.md)** Reload prefetch — TSK_012 (decode reload) 와 통합 영역 |

> **단순 입출력 검증이 아닙니다** — prefetch 트리거 시점 측정 + forward 진입 시점에 reload 완료 비율 측정 + e2e throughput 의 3 단계.

---

## 통합 결정 (2026-04-28)

TSK_009 inspection 결과 — 현재 vLLM 동작이 이미 prefetch overlap (`transfer_async` 가 별도 CUDA stream + `wait_event` 로 sync). 추가 향상 영역 (모든 cold-bearing reload / step-advance prefetch / layer-level prefetch) 은 **vLLM core block manager 통합** 영역으로 *TSK_012 (decode reload) 와 같은 hook 영역*. 단독 진행 가치 작아 **TSK_012 의 일부로 통합** 결정.

본 TST 의 (1.1 / 1.2 / 1.3) 측정은 TSK_012 진행 시점에 함께 수행. 별도 회차 없음.

---

## 1. 검증 차원

### 1.1 · prefetch 트리거 시점 측정

```python
# baseline (TSK_009 비활성, 본 commit c35585556c 의 sync 만)
trigger_time_baseline = step_t + forward_start_offset_baseline

# prefetch (TSK_009 활성)
trigger_time_prefetch = step_t + forward_start_offset_prefetch

assert trigger_time_prefetch < trigger_time_baseline, (
    "prefetch 트리거가 forward 진입보다 *이전* 시점이어야 — baseline 은 "
    "forward 진입 직전, prefetch 는 이전 step admission 시점"
)
```

host 시간 stamp (`time.perf_counter`) 로 측정.

### 1.2 · forward 진입 시점 reload 완료 비율

forward 시작 시점에 in-flight transfer event 들의 `event.query()` 호출 — True (이미 완료) 비율을 100 step 의 분포로 산출:

| 모드 | event.query() = True 비율 |
|---|---|
| baseline (sync 만, prefetch 없음) | 거의 0% (transfer 가 forward 직전 시작이라 forward 시작 시점에는 진행 중) |
| prefetch | 50%+ 목표 (이전 step 동안 transfer 가 진행되어 forward 시작 시 일부 완료) |

contract: prefetch 비율 > baseline 비율 (적어도 30 percentage point 차이).

### 1.3 · e2e throughput

같은 100 req heavy workload 의 total wall time 비교 — prefetch 가 forward latency 에서 reload 시간을 hiding 한 만큼 wall time ↓.

---

## 2. Pass / Fail 기준

| 단계 | 기준 |
|---|---|
| 1.1 | trigger_time_prefetch < trigger_time_baseline |
| 1.2 | event.query()=True 비율 ≥ 50% (prefetch) vs ≤ 10% (baseline) |
| 1.3 | wall time(prefetch) / wall time(baseline) < 0.85 (15% 이상 단축) |

세 기준 모두 통과해야 prefetch 가 throughput 에 의미 있는 기여.

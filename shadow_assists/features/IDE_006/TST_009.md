| 항목 | 값 |
|---|---|

---

## 1. invariant 정의 (사용자 framing)

| invariant | 정의 |
|---|---|
| **1 (속도)** | C (cold-tier ON, IDE_006 ON, fix v4) wall-time 이 B (cold-tier ON, IDE_006 OFF) 보다 *느려지면 안 됨*. 즉 IDE_006 의 추가 cost ≤ 0 |
| **2 (CPU 활용)** | C 회차의 cold-outcome `merged` 횟수 > 0. 즉 CPU partial 작업이 *모두* 폐기되지 않고 적어도 일부 layer 에서 GPU 가 활용 |

---

## 2. 검증 차원

### 2.1 · 단위 회귀 (`tests/v1/cpu_partial_attention/`)

전체 351 cases. fix v4 적용 후 343 passed / 8 failed. 8 fail 모두 `atol=0, rtol=0` strict 비교 (BF16 산술 비결합성, 기존 known issue) — fix v4 의 회귀 아님.

### 2.2 · prod 통합 검증 (`eval/run_prod_tsk009_validation.sh`)

3 시나리오 × 2 mode (B / C):
- input_heavy: INPUT=15360, OUTPUT=1024
- output_heavy: INPUT=1024, OUTPUT=15360
- equal: INPUT=8192, OUTPUT=8192
각 100 prompts × Llama-3.3-70B + TP=8 + max_model_len=16384.

cold-outcome counter (`_record_cold_outcome` in `flash_attn.py`) — `merged` (= `future.done()` True 시점에 hot subset paged FA + LSE merge 진입) vs `dropped` (= `future.done()` False 시점에 폐기 + paged FA full inplace) 횟수 누계.

---

## 3. 결과 (2026-04-29)

`eval/results/20260429_043734_*_tsk009_validation/`.

### 3.1 · invariant 1 (속도 — C/B wall-time)

| 시나리오 | B done (s) | C done (s) | **C/B ratio** | 판정 |
|---|---|---|---|---|
| input_heavy | 200.0 | 220.6 | **1.103×** | 부분 위반 (10.3%) |
| output_heavy | 528.1 | 573.9 | **1.087×** | 부분 위반 (8.7%) |
| equal | 314.8 | 339.2 | **1.078×** | 부분 위반 (7.8%) |

**잔여 7~10% 의 출처**: cold path setup (`reduced_query_cpu` D2H + `seqused_k.cpu()` + `num_cold_blocks.cpu()` + `cu_query_lens.cpu()` 의 GPU sync) 이 cold-bearing layer 마다 main thread block 일으킴. fix v4 가 GPU FA 호출은 1번으로 제한했지만 setup 비용 자체는 그대로.

**사용자 결정 (2026-04-29)**: **acceptable 로 인정**. 추가 fix 영역 = setup async / future thread 위임 (큰 refactor) — 본 TSK 범위 밖.

### 3.2 · invariant 2 (CPU 활용 — merged %)

| 시나리오 | merged | dropped | **merged %** | 판정 |
|---|---|---|---|---|
| input_heavy_C | 0 | 41600 | **0.00%** | 완전 위반 |
| output_heavy_C | 0 | 1600 | **0.00%** | 완전 위반 |
| equal_C | 0 | 20800 | **0.00%** | 완전 위반 |

**fundamental 원인**: Q dependency. `partial = softmax(Q · Kᵀ_cold) · V_cold` 의 Q 가 그 layer 의 QKV projection 후 결정. future submit ~ poll 시점이 *layer 안 microsecond* — CPU partial 6.4 ms 가 절대 fit 못 함.

**TSK_005 영역으로 분리** — cross-layer pipeline 으로 layer N+1 의 cold partial 을 layer N 의 hot path 진행 동시에 시작 → CPU 6.4 ms 가 *2 layer wall-time (~6 ms)* 안에 fit 가능.

## 4. Pass / Fail 판정

| invariant | 기준 | 결과 |
|---|---|---|
| 1 | C/B ≤ 1.0 | **부분 위반** (1.078~1.103×). 사용자 결정으로 acceptable |
| 2 | merged > 0 | **완전 위반** (0.00% across all scenarios). TSK_005 로 영역 이전 |

본 TST 의 *binary pass/fail* 은 invariant 1 부분 위반 + invariant 2 완전 위반이지만, 두 invariant 모두 *측정 정량* 의 의미 있는 데이터 산출 → 검증 게이트로서의 *완료* 처리.

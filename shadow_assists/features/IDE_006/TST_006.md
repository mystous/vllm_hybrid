> **단순 입출력 검증이 아닙니다** — chunk D2H 와 CPU kernel 의 stream-level 겹침을 *시간 측정* 으로 직접 입증.

---

## 1. 검증 차원

### 1.1 · 단위 — chunk D2H end vs CPU kernel start 의 시간 순서

```python
def test_chunk_kernel_starts_before_subsequent_chunk_d2h_ends():
    """N=4 chunk 로 split 했을 때, chunk 0 의 CPU kernel start 가 chunk 3
    의 D2H end 보다 *이전* — 즉 D2H 와 CPU kernel 이 stream-level 로 겹친다."""
    events = {}
    # chunk 별 D2H start/end + CPU kernel start time 측정
    ...
    assert events["kernel_start_chunk0"] < events["d2h_end_chunk3"], (
        "chunk pipelining 이 동작하면 첫 chunk kernel 이 마지막 chunk D2H "
        "끝나기 전에 시작 — sequential 이면 모든 D2H 끝난 후 kernel 시작"
    )
```

### 1.2 · 통합 — chunk 수 sweep

| chunk 수 | 측정 |
|---|---|
| 1 (현재 코드) | cold path 총 wall time |
| 2 | 같은 입력의 wall time |
| 4 | 같은 입력의 wall time |
| 8 | 같은 입력의 wall time |
| 16 | 같은 입력의 wall time |

contract: wall time(N) ≤ wall time(1). plateau 되는 N 값 산출 — README §12 Q6 의 *최적 chunk 크기* 답.

### 1.3 · Numerical equivalence

chunk N 결과가 chunk 1 결과와 BF16 5e-3 / FP16 1e-3 tolerance 내 일치 — partial LSE 합산 정확.

## 2. Pass / Fail 기준

| 단계 | 기준 |
|---|---|
| 1.1 timeline | kernel_start_chunk0 < d2h_end_chunk3 |

| 1.3 numerical | atol/rtol 통과 |

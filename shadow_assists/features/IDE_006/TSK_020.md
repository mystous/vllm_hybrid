## 1. 개요

vLLM 의 built-in **ngram speculative decoding** lever 위에서 throughput 을 극대화하고, 그 위에 **CPU 활용 lever** (CPU draft model / CPU LLM 별도 instance / multi-workload) 를 추가 결합해서 CLAUDE.md `# Objective` 의 두 축 (GPU 포함 서버 throughput 향상 + CPU 활용률 극대화) 을 동시 충족하는 영역.

## 5. 설정 (production-ready)

### 5.1 vLLM constructor

```python
LLM(
    model="meta-llama/Llama-3.3-70B-Instruct",
    tensor_parallel_size=8,
    max_model_len=16384,
    max_num_seqs=256,
    gpu_memory_utilization=0.85,
    enforce_eager=False,
    kv_cache_dtype="fp8",
    max_num_batched_tokens=8192,
    disable_log_stats=True,
    seed=0,
    speculative_config={
        "method": "ngram",
        "num_speculative_tokens": 7,
        "prompt_lookup_max": 5,
        "prompt_lookup_min": 2,
    },
)
```

### 5.2 env

```bash
export HF_HUB_OFFLINE=1
export LD_PRELOAD=/usr/lib64/libcuda.so.1
export VLLM_NGRAM_NUM_THREADS_CAP=8     # vLLM 기본 1 → 8
export VLLM_NGRAM_DIVIDE_BY_TP=0        # tp_size 로 나누지 않음 → 8 thread/rank
```

### 5.3 SUB_047 코드 변경

`vllm/v1/spec_decode/ngram_proposer.py:36-58` (commit `a243e1c9f`):

```python
cap = int(os.environ.get("VLLM_NGRAM_NUM_THREADS_CAP", "1"))
divide_by_tp = int(os.environ.get("VLLM_NGRAM_DIVIDE_BY_TP", "1"))
if cpu_count:
    self.num_numba_thread_available = max(1, min(cap, (cpu_count // 2)))
    if divide_by_tp:
        self.num_numba_thread_available //= tp_size
    self.num_numba_thread_available = max(1, self.num_numba_thread_available)
else:
    self.num_numba_thread_available = 1
```

vLLM 기본은 `min(1, cpu_count//2) // tp_size = 1 thread/rank` (TODO 미적용). SUB_047 은 이를 env-tunable 화.

## 1. TL;DR

- **Phase 1**: cold blocks 를 *진짜 GPU evict* (mirror → swap) + decode 시점 reload trigger 추가. attention 은 standard hot FA 가 모든 KV 처리. IDE_006 partition path *발화 안 함*. **D-ii 봉합** ✓ + KV pool 압박 완화.
- **Phase 2**: Phase 1 위에 **CPU partial vs PCIe reload race** 추가. `TSK_009` fix v4 의 non-blocking poll 위에 reload event 도 같이 검사. faster wins. CPU 빠른 영역에서 *reload PCIe 비용 회피* → wall-time 단축. **invariant 2 (CPU 활용)** > 0 가능.

> **`TSK_005` 가 기각된 이유 (2026-04-29) 와 본 TSK 의 의미**:
> `Q dependency + GPU 가 진짜 Q 가지면 CPU 결과 무용` dilemma 로 cross-layer 영역 폐기. CPU partial 이 *진짜 가치* 있는 영역 = cold blocks 가 *GPU 에서 진짜 free 되어 GPU 가 cold attention 못 하는* 시점 = reload 가 *필수* 인 시점. 그 시점에 CPU partial 이 *reload 의 대체* 로만 의미. 이게 본 TSK 의 Phase 2.

## 2. 발급 배경 (TSK_011 sweep 결과)

`TSK_011` §4.5 prod sweep (2026-04-28):

| 시나리오 | fallback # | worst_lp | worst_ppl | d_ii |
|---|---|---|---|---|
| 비활성 (cold-tier ON, IDE_006 OFF) | 0 | 3.428 | 0.2423 | 8/30 |
| deadline=100ms (fallback 100%) | 40 | 3.429 | 0.2299 | 9/30 |
| deadline=1000ms (fallback 0%) | 0 | 3.428 | 0.2423 | 8/30 |

→ partition / fallback 두 path *모두 cold KV 를 CPU page cache* 에서 가져옴. baseline (cold-tier OFF, 모든 KV GPU paged) 와 *ULP-level 차이* 가 logprob 에 누적 → lp ~3.43 발산. **fallback 만으로는 D-ii 봉합 불가능**.

근본 원인 = vLLM 의 cold-tier 가 *mirror* (gpu→cpu store 만, GPU 잔류) 정책. cold blocks 의 GPU/CPU 사본 사이의 BF16 noise 누적이 attention 결과 분포에 영향.

## 3. 두 framing 의 sequence diagram

### 3.1 · 현재 (TSK_012 land 전, fix v4 적용 상태)

```mermaid
sequenceDiagram
    participant Sch as Scheduler
    participant Conn as OffloadingConnector
    participant GPU as GPU
    participant CPU as CPU partial

    Sch->>Conn: prefill admission
    Conn->>GPU: store cold blocks gpu→cpu (MIRROR — GPU 잔류)
    Note over GPU,Conn: cold blocks 가 GPU + CPU 양쪽 모두 존재

    loop decode step (매 layer)
        Sch->>GPU: forward
        Note over GPU: cold-bearing 발견 — IDE_006 partition path 진입
        GPU->>+CPU: forward_partial_with_lse_async(Q, cold)
        Note over CPU: 별도 thread (~6 ms)
        GPU->>GPU: future.done() non-blocking poll
        alt done (실측 0%)
            CPU-->>-GPU: cold partial result
            GPU->>GPU: hot subset paged FA + merge_attn_states
            Note right of GPU: lp noise (CPU page cache)
        else not done (실측 100%)
            GPU->>GPU: future.cancel + paged FA full inplace
            Note right of GPU: GPU mirror 사용 (= baseline 동작)
        end
    end

    Note over Sch,CPU: 결과: invariant 1 = 1.078~1.103× 느림, invariant 2 = 0%, lp ~3.43
```

### 3.2 · Phase 1 — D-ii 봉합 only (cold 진짜 evict + decode reload, IDE_006 발화 안 함)

```mermaid
sequenceDiagram
    participant Sch as Scheduler
    participant Conn as OffloadingConnector
    participant GPU as GPU
    participant CPU as CPU partial

    Sch->>Conn: prefill admission + KV pool overflow 검사
    Conn->>GPU: store cold blocks gpu→cpu + GPU pool free (진짜 evict)
    Note over GPU,Conn: cold blocks 가 CPU 에만 존재 (KV pool 압박 완화)

    loop decode step
        Sch->>Conn: decode admission — cold-bearing seqs 식별
        Conn->>GPU: reload trigger (PCIe transfer, 별도 stream)
        Note over GPU: cold blocks PCIe 진행 중

        Sch->>GPU: forward
        GPU->>GPU: 첫 attention layer 진입 시 wait_event<br/>(reload 완료까지 대기)
        Note right of GPU: GPU 가 reload 동안 wait

        Note over GPU: cold blocks 도착 — 모든 KV GPU 잔류
        GPU->>GPU: standard paged FA full (block_table 그대로)
        Note right of GPU: IDE_006 partition path 발화 안 함<br/>baseline 동작 동등 → D-ii 봉합 ✓
    end

    Note over Sch,CPU: 결과: D-ii ✓ + KV pool 완화 + reload PCIe 비용 추가<br/>IDE_006 partition path 무용 (CPU partial 호출 안 함)
```

### 3.3 · Phase 2 — 진짜 evict + IDE_006 race (CPU partial 이 reload 대체)

```mermaid
sequenceDiagram
    participant Sch as Scheduler
    participant Conn as OffloadingConnector
    participant GPU as GPU
    participant CPU as CPU partial

    Sch->>Conn: prefill admission + KV pool overflow 검사
    Conn->>GPU: store cold blocks gpu→cpu + GPU pool free (진짜 evict)

    loop decode step
        Sch->>Conn: decode admission — cold-bearing seqs 식별

        par 두 path 동시 시작 (race)
            Conn->>GPU: PCIe reload event 시작 (별도 stream)
        and
            Sch->>+CPU: forward_partial_with_lse_async(Q, cold)
        end

        Sch->>GPU: forward
        GPU->>GPU: hot subset paged FA (cold 제외)
        GPU->>GPU: race poll (future.done vs reload event done)

        alt CPU 먼저 done
            CPU-->>-GPU: cold partial result
            GPU->>Conn: reload cancel
            GPU->>GPU: merge_attn_states(hot, CPU cold)
            Note right of GPU: wall-time 단축 (reload PCIe 회피)<br/>invariant 2 ↑
        else reload 먼저 done
            Conn-->>GPU: cold blocks GPU 도착
            GPU->>GPU: paged FA full (cold 포함)
            Note right of GPU: CPU 결과 폐기 (fix v4 패턴)
        else 둘 다 deadline 안 못 끝남
            GPU->>GPU: TSK_011 fallback path
            Note right of GPU: 안전망
        end
    end

    Note over Sch,CPU: 결과: D-ii ✓ + invariant 1 향상 가능 + invariant 2 > 0
```

## 4. Phase 1 / Phase 2 단계 비교

| 영역 | Phase 1 (D-ii 봉합 only) | Phase 2 (진짜 evict + race) |
|---|---|---|
| cold blocks 위치 | CPU only (GPU evict) | CPU only (GPU evict) |
| KV pool 압박 | 완화 ✓ | 완화 ✓ |
| decode reload trigger | 항상 발화 | race 의 한 path |
| IDE_006 partition path 발화 | 발화 안 함 | 발화 (race 의 다른 path) |
| CPU partial 호출 | 호출 안 함 | 호출 (race) |
| GPU FA 호출 | 1회 (standard paged FA full) | 1회 (race 결과에 따라 hot subset 또는 full) |
| D-ii 봉합 | YES ✓ | YES ✓ |
| invariant 1 (속도) | reload PCIe 비용으로 약간 증가 | race 로 향상 가능 (CPU 빠른 영역) |
| invariant 2 (CPU 활용) | 0% | > 0% 가능 |
| TSK_009 fix v4 코드 | dead (호출 안 됨) | 활용 (race 의 non-blocking poll) |
| TSK_011 fallback path | dead (예외 안전망만) | 재활용 (race 의 *둘 다 못 끝남* 분기) |

## 7. 검증 — `TST_012`

자세한 spec 은 [`TST_012`](TST_012.md). 핵심:

- **Phase 1**: D-ii 봉합 입증 (`worst_max_abs_logprob ≤ 0.5`, ppl_relative_diff tolerance 안)

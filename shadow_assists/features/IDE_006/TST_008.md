# TST_008 — hot/cold 분할 정책 측정 (layer-별 vs request 균일)

| 항목 | 값 |
|---|---|
| ID | `TST_008` |
| 상태 | `대기` |
| 검증 대상 | **[`TSK_008`](TSK_008.md)** hot/cold 분할 정책 |

> **단순 입출력 검증이 아닙니다** — 두 정책의 *실제 firing 빈도* 와 *e2e throughput* 을 prod 측정으로 비교 (README §12 Q4 의 답).

---

## 1. 검증 차원

### 1.1 · per-layer cold path firing 빈도 (실 측정)

각 정책 활성 시 layer 별 cold path firing count 를 stderr breadcrumb 로 capture:

```
[IDE_006/TSK_004 cold-path fired pid=... layer=N] firing_count=...
```

100 req heavy workload 의 80 layers × N steps 의 firing 분포 산출.

| 정책 | layer-별 firing 분포 |
|---|---|
| request 균일 (현재) | 모든 layer 동일 firing 빈도 |
| layer-별 독립 | 각 layer 별 다른 빈도 — early layer (low) vs later layer (high) 등 |

### 1.2 · e2e throughput

같은 100 req heavy workload 를 두 정책으로 각각 실행 — total wall time + tokens/s 비교.

### 1.3 · numerical equivalence

두 정책이 같은 attention 결과 — TST_003 의 D-ii (logprob/PPL diff) 게이트 그대로 적용.

---

## 2. Pass / Fail 기준 + 채택 결정

| 차원 | 측정 |
|---|---|
| firing 분포 | 두 정책의 layer-별 분포가 *다른지* 확인 (정책 차이가 실제로 다른 routing 만드는지) |
| e2e throughput | tokens/s 가 더 높은 정책 채택 |
| numerical | TST_003 D-ii tolerance 통과 |

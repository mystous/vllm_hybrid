---

## 1. TL;DR

- **옵션 A**: K/V 를 KV-head 단위 (compact) 로 CPU 에 저장. 연산 직전 broadcast 하여 Q-head 단위로 expand. CPU 메모리 절약, broadcast 연산 추가.
- **옵션 B**: CPU 에서 Q-head 단위로 미리 expand 된 buffer 유지. 메모리 ↑, 연산 단순.
- 본 TSK 는 두 옵션의 microbench 결과로 선택 + 코드 land.

## 3. 검증 — `TST_007`

자세한 spec 은 [`TST_007`](TST_007.md). 핵심:

- **단순 입출력 아님** — perf counter (CPU memory bandwidth `MEM_LOAD_RETIRED.L3_MISS`, L2 miss rate) + kernel wall time 의 *실 측정*.
- 두 옵션의 *cache working set* 이 L2/L3 안에 들어가는지 직접 검증 (README §12 Q3 의 답).
- 두 옵션의 numerical equivalence (BF16 5e-3 / FP16 1e-3 tolerance).

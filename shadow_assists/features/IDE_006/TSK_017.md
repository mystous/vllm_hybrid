## 1. TL;DR

NEO 의 *table-based linear interpolation* perf predictor — `linr_T(S)` (linear) / `pref_T(S)` (prefilling) / `gdec_T(N)` (GPU decode) / `cdec_T(S, N)` (CPU decode bilinear) / `lnch_T()` (kernel launch overhead) 의 5 종류 prediction. 이게 TSK_014 의 mode selection (pipelined vs sequential) 의 결정 입력.

`TablePerfPredictor` 는 startup 시 `ModelProfiler` 가 *각 (S, N) 점에서 actual forward 측정* 하여 table 채움. 이후 scheduling 의 매 iteration 에서 O(1) lookup + linear interpolation.

## 2. 변경 범위

### 2.1 · 신규 / 변경 파일

| 파일 | 변경 |
|---|---|
| `vllm/v1/core/sched/perfpredictor.py` (신규) | NEO `PerfPredictor` 추상 + `TablePerfPredictor` port. 5 종류 prediction + 1D/2D linear interpolation |
| `vllm/v1/metrics/profiler.py` (신규) | NEO `ModelProfiler` port — startup 시 profile run 으로 4 개 table 채움 |
| `vllm/v1/engine/llm_engine.py` | startup hook — `initialize()` 안 ModelProfiler 실행 |
| `vllm/v1/core/sched/sub_batch.py` | `BatchPerfData` (TSK_014 와 공유) — predictor 입력으로 `linr_T` / `gdec_T` / `cdec_T` 계산 |

### 2.2 · Profile run 의 cost / 영향

NEO 의 profile run 은 *각 (S, N) 점에서 forward 측정* — N 점이 ~수십 개. prod 환경에서 startup overhead 측정 영역:

- 각 측정 점 = 1 forward step (~수 ms ~ 수십 ms)
- 총 측정 점 ≈ `len(linr_S_list)` + `len(pref_S_list)` + `len(gdec_N_list)` + `len(cdec_S_list × cdec_N_list)` ≈ 수백
- 예상 startup overhead: < 60s (TST_017 의 acceptance)

### 3.1 · 1.5/1.6 적재 plan — engine 측 measure_fn

NEO 의 `_run_test_case` 는 artificial SubBatch 만든 후 `executor.do_one_iteration` 호출. vLLM 위에서 동일 영역:

```python
# vllm/v1/metrics/profiler.py 의 engine-side adapter
def make_measure_fn(model_runner, vllm_config) -> Callable[[str, int, int], float]:
    def measure(kind: str, p1: int, p2: int = 0) -> float:
        # kind ∈ {"linr", "pref", "gdec", "cdec", "lnch"}
        # p1 = S (batch width) / p2 = N (kv tokens)
        # 1) artificial SubBatch 빌드 — kind 별로 add_pref/add_gdec/add_cdec
        # 2) model_runner._dummy_run(...) 같은 기존 dummy run 인프라 reuse
        # 3) torch.cuda.synchronize() 전후의 wall-time 측정 (ms)
        ...
    return measure
```

vLLM 의 `gpu_model_runner.py` 에는 이미 dummy run 인프라가 있음 (`_dummy_run`, `dummy_run` 등). NEO profile 영역의 직접 reuse 가능.

### 3.2 · 1.6 의 startup wiring 위치

`LLMEngine.initialize()` 또는 `EngineCore.start()` 의 model load + KV cache alloc 직후:

```python
# vllm/v1/engine/llm_engine.py
if self.scheduler_config.enable_neo_asymmetric:
    profiler = ModelProfiler(
        predictor=self.scheduler.table_predictor,
        measure_fn=make_measure_fn(self.model_runner, self.vllm_config),
    )
    profiler.run()
    self.scheduler.predictor = self.scheduler.table_predictor
    self.scheduler.neo_scheduler.predictor = self.scheduler.table_predictor
```

(현재 `NeoSchedulerAdapter.__init__` 에서 `table_predictor` 가 init 됐지만 `predictor` 는 ZeroPerfPredictor — swap 영역.)

### 3.3 · 1.7 caching 전략

- 동일 `(model_name, gpu_arch, dtype, tp_size, max_num_seqs, block_size)` tuple 에 대한 table 은 *deterministic* (workload-independent measurement).
- hash key = SHA-256(serialized tuple)
- save path = `~/.cache/vllm/neo_perfpredictor/<hash>.json`
- cold start (table 없을 때) 만 ModelProfiler 실행 (60s overhead). warm start 는 즉시 load.

```python
# vllm/v1/core/sched/perfpredictor.py
class PerfPredictor:
    def get_linr_T(self, S: int) -> float: ...   # linear (FFN/proj) 시간
    def get_pref_T(self, S: int) -> float: ...   # prefilling 시간
    def get_gdec_T(self, N: int) -> float: ...   # GPU decode 시간
    def get_cdec_T(self, S: int, N: int) -> float: ...  # CPU decode (bilinear)
    def get_lnch_T(self) -> float: ...           # launch overhead

class TablePerfPredictor(PerfPredictor):
    def __init__(self, engine_config):
        # x 값 list — 2 의 거듭제곱 + 보강 점
        self.linr_S_list = list(range(1, 512)) + [2**i for i in range(9, ...)]
        # ... 다른 list 들도 정의

        # Lower-bound index 미리 계산 (O(1) lookup)
        self.linr_S_lb_idx = self._get_lb_idx_list(self.linr_S_list)
        # ...

    def _interp(self, x, x0, x1, y0, y1):
        return y0 + (y1 - y0) * (x - x0) / (x1 - x0)

    def _interp_1d(self, x, xs, ys, lb_idx):
        if x == 0: return 0.0
        idx = lb_idx[x]
        if idx == 0 or x == xs[idx]: return ys[idx]
        return self._interp(x, xs[idx-1], xs[idx], ys[idx-1], ys[idx])

    def get_cdec_T(self, S, N):
        # S 차원 보간 후 N 차원 보간 (bilinear)
        ...
```

`ModelProfiler` 의 startup 측정 ([`NEO_code_deepdive.md`](NEO_code_deepdive.md) §6.4):

```python
# vllm/v1/metrics/profiler.py
class ModelProfiler:
    def init_profile_tables(self, block_manager):
        # 4 종류 table 채우기
        self.pp.linr_S_list, self.pp.linr_T_list = self._profile_linr(...)
        self.pp.pref_S_list, self.pp.pref_T_list = self._profile_pref(...)
        self.pp.gdec_N_list, self.pp.gdec_T_list = self._profile_gdec(...)
        self.pp.cdec_S_list, self.pp.cdec_N_lists, self.pp.cdec_T_lists = \
            self._profile_cdec(...)

    def _run_test_case(pref_lens, gdec_lens, cdec_lens, nwarmup=2, nrepeat=3):
        # artificial test case 로 SubBatch 만든 후 actual forward 실행
        nbatches = len(pref_lens)  # 1 (sequential) 또는 2 (pipelined)
        batches = [SubBatch() for _ in range(nbatches)]
        # ... add_pref / add_gdec / add_cdec
        args = self.bm.prepare(batches, [], [])

        for i in range(-nwarmup, nrepeat):
            if i == 0: self.executor.turn_on_perf_monitor()
            self.executor.do_one_iteration(batches, *args)

        return self.executor.turn_off_perf_monitor_and_flush_results()
```

핵심 — 두 변종:
- `_run_test_case_seq` = 단일 SubBatch (sequential 측정)
- `_run_test_case_pip_same` = 두 동일 SubBatch (pipelined 측정)

총 측정 점 ≈ 수백, nwarmup=2 + nrepeat=3 → startup 시간 acceptance < 60s (TST_017).

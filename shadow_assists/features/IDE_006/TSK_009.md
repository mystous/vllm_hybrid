# TSK_009 — Cold-path non-blocking dispatch (CPU 가 GPU 작업 방해 없이 best-effort 활용)

| 항목 | 값 |
|---|---|
| ID | `TSK_009` |
| 상태 | `활성 (fix v4 land — invariant 1 부분 충족 7~10% 잔여, invariant 2 0% — TSK_005 영역으로 분리)` (2026-04-29) |
| 부모 PLN | [`PLN_001`](PLN_001.md) |
| 목적 | IDE_006 partition path 의 *non-blocking dispatch* 정착. CPU partial 은 별도 thread 에서 진행하여 GPU main thread 방해 없음. CPU 결과가 GPU 가 필요한 시점까지 도착해 있으면 best-effort 활용, 미도착이면 *결과 폐기* + GPU 가 paged FA full 으로 자기 계산. *알고리즘적으로 baseline 보다 느릴 수 없음* 의 invariant 충족 |
| 검증 게이트 | [`TST_009`](TST_009.md) |

> **사용자 framing (2026-04-29)** — "CPU 가 GPU 의 작업을 절대로 방해해서는 안됨. Cold Tier 가 host 에 오는 순간부터 partial attention 가능 시점 기다리다 바로 진행. partial attention 이 GPU 가 필요할 때까지 준비 안 되면 그냥 폐기, 이때는 GPU 가 스스로 계산하는 거야". 본 TSK 의 algorithmic invariant.

---

## 1. TL;DR

- **무엇을 하는가**: cold-bearing layer 에서 CPU partial attention 을 별도 thread (`ThreadPoolExecutor`) 에 submit. layer 진입 시 *non-blocking* `future.done()` 으로 검사. done → hot subset paged FA + LSE merge. not done → CPU 결과 폐기 + paged FA full inplace + return.
- **왜 이 형태**: GPU 의 main thread 는 CPU 결과를 *기다리지 않는다*. GPU FA 호출 = *항상 1번* (done = hot subset 1회 / not done = full 1회). cold-tier ON + IDE_006 OFF baseline 동작과 비교 시 IDE_006 의 추가 *FA call cost = 0*.
- **결과 (2026-04-29 prod 6 회차 검증)**: invariant 1 (속도) 부분 충족, invariant 2 (CPU 활용) fundamental 위반. 본문 §3 참조.

## 2. 사용자 framing 의 4 invariant

| # | invariant | fix v4 충족 |
|---|---|---|
| 1 | CPU 가 GPU 의 작업을 절대 방해 안 함 | `future.done()` non-blocking poll. main thread block 0 |
| 2 | Cold Tier host 도착 순간부터 partial 가능 시점 기다리다 바로 진행 | Q dependency 로 layer 진입 후에만 시작. 진정한 *사전* 처리는 cross-layer (TSK_005) 영역 |
| 3 | partial attention 이 GPU 필요할 때까지 준비 안 되면 폐기 | `future.cancel()` + 결과 미사용 |
| 4 | 폐기 시 GPU 가 스스로 계산 (= cold-tier ON IDE_006 OFF baseline 동작 동등) | not done 분기에서 paged FA full inplace 호출 |

## 3. fix v4 동작 정의 + prod 검증

### 3.1 · `hot_cold_attention` 흐름 (fix v4)

```
fail-closed pre-check                 ← prefill chunk + cold blocks 검사 (existing)
if max_num_cold_blocks == 0:
    flash_attn_varlen_func(... block_table=block_table, out=output)
    return                             ← cold-less fast path

cold path setup                        ← reduced_query_cpu D2H, reduced_cbi etc.

if deadline_s is None:
    cold_output, cold_lse = forward_partial_with_lse(...)   ← sync mode (legacy)
    hot_output, hot_lse = _call_hot_subset_paged_fa()
    → partition merge tail
else:
    future = forward_partial_with_lse_async(...)
    if future.done():                                       ← 사용자 framing
        cold_output, cold_lse = future.result()
        _record_cold_outcome(merged=True)
        hot_output, hot_lse = _call_hot_subset_paged_fa()
        → partition merge tail (GPU FA = hot subset 1회)
    else:
        future.cancel()
        _record_cold_outcome(merged=False)
        flash_attn_varlen_func(... block_table=block_table, out=output)
        return                                              (GPU FA = full 1회)
```

### 3.2 · prod 검증 (2026-04-29) — 100 prompts × Llama-3.3-70B + TP=8

3 시나리오 (input + output = 16384) × 2 mode (B = cold-tier ON IDE_006 OFF / C = fix v4 적용):

| 시나리오 | input/output | B done (s) | C done (s) | **C/B** | merged % |
|---|---|---|---|---|---|
| input_heavy | 15360/1024 | 200.0 | 220.6 | **1.103×** | 0.00% |
| output_heavy | 1024/15360 | 528.1 | 573.9 | **1.087×** | 0.00% |
| equal | 8192/8192 | 314.8 | 339.2 | **1.078×** | 0.00% |

**결과 디렉토리**: `eval/results/20260429_043734_*_tsk009_validation/`.

### 3.3 · invariant 충족 정량

- **invariant 1 (속도)**: B 대비 7~10% 잔여 cost — partial 위반. 잔여 cost 의 출처 = cold path setup (D2H reduced query / `seqused_k.cpu()` / `num_cold_blocks.cpu()` 의 GPU sync) 비용. fix v4 가 GPU FA 호출 횟수는 1번으로 제한했지만 setup 의 GPU sync block 자체는 그대로. 사용자 결정 (2026-04-29): **acceptable 로 인정**.
- **invariant 2 (CPU 활용)**: 모든 시나리오 merged=0/dropped=N (**0.00%**). CPU partial 6.4 ms / GPU hot 0.6 ms 의 fundamental gap + future submit ~ poll 시점 microsecond → CPU 가 절대 GPU 의 busy window 안에 fit 못 함. **TSK_005 (cross-layer) 영역으로 분리**.

### 3.4 · 이전 fix 시도 비교

| fix 버전 | 동작 | input_heavy C/B |
|---|---|---|
| (revert 후 origin) deadline + fallback | `future.result(timeout)` block + paged FA full *추가* 호출 | 5.6× 초과 |
| fix v3 (non-blocking poll + fallback) | `future.done()` 검사 + `_fallback_full_fa_paged` 추가 호출 → GPU FA 2회 | 1.142× |
| **fix v4 (non-blocking poll + paged FA inplace)** | hot path 호출을 done 분기로 *지연*. not done 시 GPU FA full inplace → GPU FA 1회 | **1.103×** ← 적용 |

## 5. invariant 2 미충족의 fundamental 원인 + 후속 영역

`partial = softmax(Q · Kᵀ_cold) · V_cold`. **Q dependency** 가 fundamental:
- Q 는 그 layer 의 QKV projection 후 결정 (layer attention 진입 직전)
- future submit 시점 ~ poll 시점 사이가 *layer 안 setup time* (microsecond) → CPU 6.4 ms 가 fit 못 함
- 즉 invariant 2 는 *layer-안 framing 으로는 충족 불가능*

### 5.1 · 후속 영역 후보 (사용자 결정 2026-04-29)

| 영역 | model-agnostic | 효과 | 작업량 |
|---|---|---|---|
| **TSK_005 cross-layer pipeline** | NO (모델별 hook 위치) | **큼** (~50%+ merged 가능) | 큼 |
| (B) cross-step prefetch (decode token N+1 의 layer 0 만) | YES | 매우 작음 (~1.25%, 1/80 layer) | 중간 |
| (C) speculative Q (추정 → cold partial 미리) | YES | 중간 (hit rate 의존) | 큼 + 정확도 risk |
| (D) async D2H (future thread 가 GPU tensor 받아 D2H) | YES | 매우 작음 (D2H ms 단위) | 작음 |
| (F) CPU kernel 단축 | YES | 한계 도달 (TSK_007 옵션 A 채택) | 작음 |

**결정**: invariant 2 의 의미 있는 향상 위해서는 **TSK_005 (cross-layer)** 가 유일. model-specific 한계는 *공통 abstraction layer* (attention backend 의 `prefetch_next_layer_cold` 같은 hook) + 모델별 forward 의 hook 호출로 부분 완화. TSK_005 의 후속 진행 영역.

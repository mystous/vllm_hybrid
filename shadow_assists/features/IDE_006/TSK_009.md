## 1. TL;DR

vLLM 의 `OffloadingConnectorScheduler.update_state_after_alloc()` (`scheduler.py:247-292`) 가 prefill admission 시점에 `prepare_load + GPULoadStoreSpec` 으로 reload 작업을 spec 화. 그 후 `start_load_kv` 가 *forward 직전* 에 transfer 시작. 즉 reload window 가 *forward 와 동시* — forward 가 reload 끝까지 wait (본 commit 의 sync).

본 TSK: reload 시작 시점을 *이전 step 의 admission* 또는 *그 이전* 으로 옮겨 forward 와 *prefetch overlap*. 즉 reload 가 forward 가 시작될 때 이미 *진행 중* 이거나 *완료* 상태.

---

## 2. 변경 범위

vLLM core 의 KVConnector lifecycle 변경 영역. `OffloadingConnector.start_load_kv` 의 *호출 시점* 을 forward 직전이 아니라 *이전 step 의 끝* 또는 *별도 thread* 로 옮김.

| 단계 | 산출물 |
|---|---|
| 2.1 prefetch hook 설계 — 어느 step lifecycle 자리에 trigger | `vllm/distributed/kv_transfer/kv_connector/v1/offloading_connector.py` + scheduler |
| 2.2 prefetch 시작 시점에서 transfer event 를 forward 시점에 대기 | 본 commit 의 `wait_event` 와 호환 — sync 위치는 그대로, trigger 시점만 앞당김 |
| 2.3 prefetch 효과 측정 | TST_009 |

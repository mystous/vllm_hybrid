## 1. 검증 차원

### 1.1 · 회귀 — IDE_006 의 기존 단위 테스트 (binding)

LSE-반환 인터페이스 제거 외 회귀 없음:

| 기존 테스트 | 영향 | 기준 |
|---|---|---|
| `test_python_ref_vs_portable.py` | LSE 비교 부분 제거, output 비교 그대로 | 통과 (수치 동등성 보존) |
| `test_avx512_cross_check.py` | 동일 | 통과 |
| `test_amx_cross_check.py` | 동일 | 통과 |
| `test_tsk010_sub_batched.py` | sub-batching 인프라 그대로 | 통과 |
| `test_numa_aware.py` | NUMA bind 그대로 | 통과 |

### 1.2 · 정확도 — CPU sub-batch attention vs GPU full attention reference (binding)

NEO 식 architecture 의 *동등 입력* 에 대해:

| 검증 항목 | 기준 |
|---|---|
| 동일 (Q, K, V, block_table) 의 GPU paged FA reference 와 CPU kernel 출력 비교 | BF16 ~5e-3 / FP16 ~1e-3 tolerance (TSK_004 와 동일 수준) |

### 1.3 · Throughput — IDE_006 의 AVX-512/AMX vs NEO 의 ISPC AVX2

prod target (Xeon SPR + Llama-70B + TP=8) 에서:

| Kernel | 측정 |
|---|---|
| IDE_006 AVX-512 | 평균 cdec_T (per request × seq_len) |
| IDE_006 AMX | 동일 |
| NEO `pacpu` ISPC AVX2 (참고용 — NEO 자체 build 시) | 동일 |

**Pass 기준**: IDE_006 의 AMX kernel 이 NEO 의 ISPC AVX2 kernel 보다 prod 환경에서 ≥ 1.5× 빠름 (= AVX-512/AMX 우위 입증).

비교 측정은 *(선택)* — NEO 자체 build 환경 (TST_013 §3) 와 동행 영역.

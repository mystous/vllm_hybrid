## 1. TL;DR

vLLM 의 *mirror* 정책 (cold blocks 가 GPU + CPU 양쪽 잔류) 을 NEO 의 *request 단위 exclusive ownership* 으로 변경. 한 request 의 KV 는 GPU 또는 CPU 한 쪽에 *exclusive* 로 존재. swap-in/out 은 *atomic* — source 에서 free + dest 에 alloc.

이 정책 변경이 NEO 의 *batch 확장* 을 가능하게 하는 fundamental 영역. mirror 라면 GPU 공간이 cold blocks 로 점유되어 batch size 못 키움.

## 2. 변경 범위

### 2.1 · 신규 / 변경 파일

| 파일 | 변경 |
|---|---|
| `vllm/v1/kv_offload/spec.py` | offload spec 에 `policy: Literal["mirror", "exclusive"]` 추가 (기본 mirror, NEO 식 적용 시 exclusive) |
| `vllm/distributed/kv_transfer/kv_connector/v1/offloading_connector.py` | exclusive policy 의 swap-in/out lifecycle |
| `vllm/v1/kv_offload/worker/cpu_gpu.py` | `transfer_async` 의 exclusive policy 옵션 — `_initiate_swap` 의 atomic (source free + dest alloc) 패턴 |
| `vllm/v1/core/kv_cache_manager.py` | DeviceBlockManager 추상 신규 — GPU/CPU 별 free block table + per-request ownership tracking |
| `vllm/v1/core/block_manager.py` (신규) | NEO `BlockManager` port — `_alloc_blocks_for_batch` / `_free_blocks_of_requests` / `_initiate_swap` |

### 2.2 · vLLM 기존 mirror 와의 호환

mirror policy 는 *config flag 로 선택 가능* — 기본은 mirror (기존 user 무회귀), `--kv-cache-policy=exclusive` 로 NEO 식 활성. TSK_014 의 scheduler 가 *exclusive* 를 가정하므로 NEO 식 적재 시 함께 활성.

## 4. 알고리즘 reference

[`NEO_code_deepdive.md`](NEO_code_deepdive.md) §5.

핵심 invariant — *exclusive*:

```python
# 어느 시점이든:
for req in all_requests:
    has_gpu = req in gpu_block_manager.allocated
    has_cpu = req in cpu_block_manager.allocated
    assert has_gpu ^ has_cpu  # XOR — 정확히 한 쪽
```

`_initiate_swap` 의 atomic:

```python
def _initiate_swap(reqs, is_swap_out):
    src = gpu_block_manager if is_swap_out else cpu_block_manager
    dst = cpu_block_manager if is_swap_out else gpu_block_manager
    src_pids = src.free(reqs)              # source 의 block 회수
    dst_vids, dst_pids = dst.alloc(reqs)   # dest 에 alloc
    return src_pids, dst_vids, dst_pids
```

→ swap 의 *physical transfer* (PCIe) 는 별도 worker thread 에서. 본 TSK 의 영역은 *block table 의 ownership 갱신 atomic*.

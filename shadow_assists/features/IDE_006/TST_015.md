---

## 1. 검증 차원

### 1.1 · 단위 — exclusive invariant (binding)

| 검증 항목 | 기준 |
|---|---|
| 어느 시점이든 한 request 의 KV 가 GPU/CPU 한 쪽에만 | `has_gpu ^ has_cpu == True` (XOR) |
| `_initiate_swap` 의 atomic | swap 전후로 invariant 보존 (race condition 영역 0) |
| split-aware (`nsplits = 1 + extra_layer_for_cprf`) | CPU prefill 이 split 1, GPU prefill / decode 가 split 0 |

### 1.2 · 회귀 — mirror policy 무회귀

`--kv-cache-policy=mirror` (기본) 이면 vLLM 기존 동작과 *bit-exact* 동등:

| 시나리오 | 기준 |
|---|---|
| 기존 OffloadingConnector user (Llama-70B + TP=8 + cold-tier ON, IDE_006 OFF) | wall-time 기존 ± 5% |
| 단위 테스트 `tests/v1/kv_offload/` | 모두 통과 |

### 1.3 · capacity — KV pool overflow 영역

`--kv-cache-policy=exclusive` 활성 시 vanilla mirror 대비 *동시 in-flight request 수* 비교:

| 시나리오 | 측정 |
|---|---|
| max-prompts=100 × 16384 tokens (= 1.64M) > KV pool 1.22M (Llama-70B + TP=8) | mirror = preempted/rejected 수 vs exclusive = preempted/rejected 수 |
| 동시 in-flight | mirror N_max vs exclusive N_max |

**Pass 기준**: exclusive 의 동시 in-flight ≥ mirror 의 동시 in-flight (= NEO 의 capacity 가치 영역 입증).

---

## 2. Pass / Fail 기준

| 단계 | 기준 |
|---|---|
| 1.1 binding | exclusive invariant 단위 테스트 통과 |
| 1.2 회귀 | mirror policy 무회귀 |
| 1.3 capacity | exclusive ≥ mirror (동시 in-flight) |

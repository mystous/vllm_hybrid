## 1. TL;DR

현재 정책: request 단위로 cold prefix 길이가 결정 (`OffloadingManager.lookup()` 의 prefix-only contiguous count, `PLN_001_TSK_002_01` Decision 1). 모든 layer 가 동일 cold blocks 사용.

대안 정책: layer 별로 cold ratio 다름. 예 — early layer 는 더 많은 hot, later layer 는 더 많은 cold. layer attention pattern 의 *temporal locality* 차이 활용.

본 TSK 는 두 정책의 e2e throughput 비교 + 채택안 결정.

---

## 2. 변경 범위

| 단계 | 산출물 |
|---|---|
| 2.1 layer-별 정책의 metadata schema 확장 | `AttentionMetadata.num_cold_blocks` 를 `[num_seqs, num_layers]` 로 확장 (현재는 `[num_seqs]`) |
| 2.2 scheduler populate 로직 | `OffloadingConnectorScheduler.build_connector_meta` 에서 layer-별 결정 |

# TST_011 — Speculative cold + GPU fallback 동작 검증

| 항목 | 값 |
|---|---|
| ID | `TST_011` |
| 상태 | `대기` |
| 검증 대상 | **[`TSK_011`](TSK_011.md)** Speculative cold + GPU fallback |
| 매핑 IDE_006 진입 조건 | **(g)** "throughput 하한 보장" — 본 TST 가 fallback 발동 빈도 + best/worst case throughput 분포 측정으로 충족 판정 |

> **단순 입출력 검증이 아닙니다** — deadline timing / fallback 발동 빈도 / throughput 분포의 *실 측정*. IDE_006 활성이 GPU only 보다 *느릴 수 없음* 을 직접 입증.

---

## 1. 검증 차원

### 1.1 · cold deadline 도달 / 미도달 시점 측정

매 layer 의 cold issue ~ deadline check 까지의 host time stamp 와 cuda event query 결과를 layer 단위로 capture:

```
layer_idx | cold_issue_t | merge_t | cold_ready_at_merge | dl_remaining_ms
```

100 step × 80 layer 의 분포 산출. ready 비율 = (ready=True) / total layer attention calls.

### 1.2 · fallback 발동 빈도 vs workload 특성

| workload axis | 측정 |
|---|---|
| cold ratio (offloaded 비율) | fallback 빈도 곡선 |
| context length | 같음 |
| batch size | 같음 |
| layer index | layer 별 fallback 분포 (early vs later layer) |

### 1.3 · **Throughput 하한 보장** (핵심 게이트)

같은 workload 를 두 모드로 측정:

| 모드 | 측정 |
|---|---|
| GPU only (`enable_cpu_partial_attention=False`) | total wall time (baseline) |
| IDE_006 활성 (TSK_011 적용) | 같은 workload 의 wall time |

**Pass 기준**: 모든 workload 에서 wall_time(IDE_006 활성) ≤ 1.05 × wall_time(GPU only) — 즉 IDE_006 활성이 5% 이상 느리지 않음. best case (cold path overlap 충분) 에서는 wall_time(IDE_006) < wall_time(GPU only).

이게 *throughput 하한 보장* 의 정량적 정의. 충족 못하면 README §3.1 3차 재정의의 약속이 깨진 것 — TSK_011 reopen.

### 1.4 · Numerical equivalence

partition 합산 layer 와 fallback layer 가 섞인 forward 의 출력이 GPU only 와 분포 동등 — TST_003 의 D-ii (logprob/PPL diff) 게이트 그대로 적용.

## 2. Pass / Fail 기준

| 단계 | 기준 |
|---|---|
| 1.1 timing | layer_idx 별 ready 비율 분포 산출 — Pass/Fail 없음 (data) |
| 1.2 fallback 빈도 | workload axis 별 곡선 산출 — Pass/Fail 없음 (data) |
| **1.3 throughput 하한** | **wall_time(IDE_006) ≤ 1.05 × wall_time(GPU only)** — 모든 측정 시나리오 |
| 1.4 numerical | TST_003 D-ii tolerance |

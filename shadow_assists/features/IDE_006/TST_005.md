## 1. 검증 차원 (3 단계)

### 1.1 · 단위 — CUDA event timeline 으로 cross-layer overlap 직접 입증

```python
@cuda_required
def test_cross_layer_overlap_in_event_timeline():
    """layer N 의 cold work end_event 가 layer N+1 의 hot FA start_event
    보다 *늦게* fire 되어야 cross-layer overlap 이 일어난 것."""
    events = {}
    # layer N 의 cold issue 시점 + cold end_event capture
    # layer N+1 의 hot FA start_event capture
    ...
    # 핵심 contract:
    assert events["cold_N_end"].elapsed_time(events["hot_N+1_start"]) <= 0, (
        "layer N cold 가 layer N+1 hot FA 시작 이전에 끝났으면 cross-layer "
        "overlap 이 안 일어난 것 — sequential 과 동일"
    )
    # overlap 영역 측정
    overlap_ms = events["cold_N_end"].elapsed_time(events["hot_N+1_start"])
    assert overlap_ms > 0  # cold N 이 hot N+1 과 시간적으로 겹침
```

### 1.2 · 통합 — wall time 비교 (sequential vs layer-async)

| 모드 | 측정 |
|---|---|
| baseline (TSK_005 비활성, sequential) | 80 layers 의 cold path 총 wall time |
| layer-async (TSK_005 활성, K=1) | 같은 batch 의 총 wall time |
| layer-async K=2/3/4 | depth sweep — README §12 Q6 의 답 |

contract: layer-async wall time **<** baseline (cross-layer overlap 만큼 짧아짐).

### 1.3 · CPU thread state — layer N 의 cold 가 background 에서 진행 중인지

`/proc/<pid>/task/<tid>/status` 의 State + cpu utilization. layer N+1 의 hot FA 가 launch 된 시점에:
- main thread: GPU launch / Python 코드 (state R 또는 S in epoll)
- worker thread: layer N 의 cold C++ kernel 진행 (state R, wchan=0)

이 두 thread 가 *동시 R 상태* 임을 5 ms 단위 sampling 으로 확인.

## 2. Pass / Fail 기준

| 단계 | 기준 |
|---|---|
| 1.1 단위 timeline | overlap_ms > 0 (cross-layer overlap 발생) |
| 1.2 wall time | layer-async wall time / baseline wall time < 0.9 (10% 이상 단축) |
| 1.3 thread state | main + worker 동시 R 상태 sampling 비율 > 50% |

세 기준 모두 통과해야 README §9 (g) overlap 가능 게이트 충족.

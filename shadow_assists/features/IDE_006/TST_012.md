## 1. 의도

`TSK_011` sweep 으로 입증된 *fallback 만으로 D-ii 봉합 불가능* 의 후속 검증. `TSK_012` 가 decode 시점에 cold blocks 를 GPU paged cache 로 reload 하여 baseline 과 동일 source 를 만들면, D-ii (per-position logprob max abs + 시퀀스 PPL relative diff) 가 tolerance 안으로 들어와야 한다.

---

## 2. 환경 — TSK_011 sweep 회차와 동일

```
모델:        meta-llama/Llama-3.3-70B-Instruct
TP:          8
max-prompts: 30 (prompt 6 / prompt 8 등 발산 prompts 포함 보장)
max-tokens:  16
logprobs:    1
baseline:    eval/envs/vllm_original_long_ctx.env
split_on:    eval/envs/ide006_cold_kv_split_on_long_ctx.env (+ TSK_012 적용)
wrapper:     eval/run_prod_quick_tst003.sh
```

TSK_011 sweep 의 baseline / split_on 결과를 직접 비교 가능한 formate.

## 3. 통과 기준

### 3.1 — D-ii (binding)

| 메트릭 | TSK_011 sweep 결과 | TSK_012 목표 |
|---|---|---|
| worst_max_abs_logprob | 3.428 (비활성) / 3.429 (fb=100%) / 3.428 (fb=0%) | **≤ atol=0.5** |
| worst_ppl_relative_diff | 0.2423 / 0.2299 / 0.2423 | **≤ rtol=0.1** |
| d_ii_pass count | 8/30 / 9/30 / 8/30 | **≥ 27/30** (90%) |

### 3.2 — D-i (informational)

| 메트릭 | TSK_011 sweep 결과 | TSK_012 목표 |
|---|---|---|
| worst_diverging_tokens | 14 | **≤ 10** (max_diverging_tokens) |
| d_i_pass count | 24/30 | **≥ 28/30** |

### 3.3 — reload event timeline (a)

decode step 에 cold blocks 의 reload trigger 시점 (host timestamp) + completion 시점 (CUDA event query) 분포를 기록. *forward 진입 시점에 reload 가 완료되어 있는 비율* 이 ≥ 90% 여야 함 — 그렇지 않으면 race window 가 열려 정확도 회귀.

### 3.4 — throughput tradeoff (c)

`split_on done in ` 시간이 baseline 대비 얼마나 늘어나는지. 허용 한계는 본 TST 의 spec 은 아니지만 정량 기록:

```
baseline   (split off):                  ~110s
TSK_011 split_on (deadline=100ms / fb=100%): 187s (+70%)
TSK_011 split_on (deadline=1000ms / fb=0%):  216s (+96%)
TSK_012 split_on (decode reload):              ?  (목표: 비교 데이터 수집)
```

`TSK_005` (cross-layer pipeline) 적용 후 회복 가능성 — 별도 측정.

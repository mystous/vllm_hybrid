## 1. 의도

`TSK_011` sweep 으로 입증된 *fallback 만으로 D-ii 봉합 불가능* 의 후속 검증. **TSK_012 의 두 phase 별로 검증 차원이 다름** (2026-04-29 갱신):

- **Phase 1 (D-ii 봉합 only)**: cold blocks 진짜 evict + decode reload trigger. partition path 발화 안 함. *D-ii 봉합* 단일 목표 검증.
- **Phase 2 (진짜 evict + IDE_006 race)**: Phase 1 위에 race 메커니즘 통합. *invariant 1 (속도) + invariant 2 (CPU 활용) + D-ii* 동시 검증.

TSK_005 기각 후 (`Q dependency dilemma`) 본 TST 가 IDE_006 의 *진짜 가치 영역 검증 게이트* 의 위치를 가짐.

### 3.3 · Phase 1 — reload event timeline

decode step 에 cold blocks 의 reload trigger 시점 (host timestamp) + completion 시점 (CUDA event query) 분포 기록. *forward 진입 시점에 reload 가 완료되어 있는 비율* 이 ≥ 90% 여야 함 — 그렇지 않으면 race window 가 열려 정확도 회귀.

### 3.4 · Phase 1 — throughput tradeoff

```
baseline (cold-tier OFF):                       ~164s (TSK_009 fix v4 검증 기준)
TSK_011 split_on (deadline=100ms / fb=100%):    187s (+14%)
TSK_011 split_on (deadline=1000ms / fb=0%):     216s (+32%)
TSK_009 fix v4 (cold-tier ON, IDE_006 OFF B):   200s (+22% — vLLM 자체 cold-tier)
TSK_009 fix v4 (cold-tier ON, IDE_006 ON C):    220s (+34%)
Phase 1 split_on (decode reload, partition off): ?  목표 = B 와 동등 (+22%)
```

Phase 1 은 partition path 발화 안 함 → wall-time = B (cold-tier ON, IDE_006 OFF) + reload PCIe 비용. 단 cold blocks 가 진짜 evict 되면 KV pool 압박 완화 → 다른 영역에서 회복 가능.

### 3.5 · Phase 2 — invariant 1 (속도) + invariant 2 (CPU 활용)

**TSK_009 fix v4 의 검증 영역을 그대로 재사용 + 개선 측정**:

| 메트릭 | TSK_009 fix v4 | Phase 2 목표 |
|---|---|---|
| C/B (input_heavy) | 1.103× | **≤ 1.05×** (5% 안 또는 향상) |
| C/B (output_heavy) | 1.087× | **≤ 1.05×** |
| C/B (equal) | 1.078× | **≤ 1.05×** |
| merged % (input_heavy) | 0.00% | **> 30%** (CPU 가 reload 보다 빠른 cell 영역) |
| merged % (output_heavy) | 0.00% | **> 30%** |
| merged % (equal) | 0.00% | **> 30%** |

### 3.6 · Phase 2 — race winner 분포

`_record_cold_outcome` counter 의 *3 way* 확장 — `merged_cpu` (CPU 빠름) / `merged_reload` (reload 빠름) / `dropped_deadline` (둘 다 늦음). 분포로 race 메커니즘이 정상 동작 확인.

### 3.7 · Phase 2 — D-ii (Phase 1 동등)

Phase 2 도 race 의 어느 분기든 baseline 동등. D-ii 봉합 ≥ 27/30 유지.

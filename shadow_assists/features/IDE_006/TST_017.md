## 1. 검증 차원

### 1.1 · 단위 — table-based interpolation 정확성 (binding)

| 검증 항목 | 기준 |
|---|---|
| `_interp` 두 점 보간 | exact (analytical) |
| `_interp_1d` 1D 보간 | exact + boundary check |
| `get_cdec_T` bilinear | exact (S, N 모두 보간) |
| `_get_lb_idx_list` 의 lower-bound 인덱스 | 모든 정수 x 에 대해 정확 |

### 1.2 · prod 정확도 — predicted vs actual layer time (binding)

prod target (Xeon SPR + H100×8 + Llama-70B + TP=8) 에서 startup profile run 후:

| 측정 | 기준 |
|---|---|
| `linr_T(S)` predicted vs actual | 오차 ≤ 10% (mean) |
| `pref_T(S)` predicted vs actual | 오차 ≤ 10% |
| `gdec_T(N)` predicted vs actual | 오차 ≤ 10% |
| `cdec_T(S, N)` predicted vs actual | 오차 ≤ 15% (CPU 변동 큼) |

오차 > 10% 면 *table 의 측정 점 부족* 또는 *prod 환경 noise* 영역. table 점 추가 또는 multi-run averaging 영역.

### 1.3 · startup overhead

profile run 의 wall-time 측정:

| 항목 | 기준 |
|---|---|
| 총 startup overhead | < 60 s |
| 측정 점 수 | ~수백 |
| 측정 점당 시간 | < 200 ms |

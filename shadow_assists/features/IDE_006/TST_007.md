# TST_007 — GQA broadcast 옵션 A vs B microbench

| 항목 | 값 |
|---|---|
| ID | `TST_007` |
| 상태 | `대기` |
| 검증 대상 | **[`TSK_007`](TSK_007.md)** GQA K/V broadcast 옵션 결정 |

> **단순 입출력 검증이 아닙니다** — Linux perf counter 로 *실제 CPU memory bandwidth + cache hit/miss* 를 측정. 두 옵션의 working set 이 L2/L3 안에 들어가는지 *직접* 확인 (README §12 Q3).

---

## 1. 검증 차원

### 1.1 · perf counter 측정

prod 머신 (Xeon SPR+) 에서 `perf stat -e` 로 측정:

| event | 측정 의미 |
|---|---|
| `MEM_LOAD_RETIRED.L1_HIT` | L1 cache hit 빈도 |
| `MEM_LOAD_RETIRED.L2_HIT` | L2 cache hit 빈도 |
| `MEM_LOAD_RETIRED.L3_HIT` | L3 cache hit 빈도 |
| `MEM_LOAD_RETIRED.L3_MISS` | L3 miss → DRAM 접근 빈도 |
| `cycle_activity.stalls_l3_miss` | L3 miss 로 인한 stall cycles |
| `mem_inst_retired.all_loads` | total loads (분모) |

옵션 A (compact + broadcast) vs 옵션 B (pre-expand): 같은 batch / context 로 두 번 실행, perf counter 비교.

### 1.2 · kernel throughput

같은 batch / context 로 cold partial attention kernel wall time 측정. tokens/s 산출.

### 1.3 · Numerical equivalence

두 옵션이 같은 attention 결과 — BF16 5e-3 / FP16 1e-3 tolerance.

---

## 2. Pass / Fail 기준 + 채택 결정

| 차원 | 측정 |
|---|---|
| L3 miss rate | A vs B 의 `L3_MISS / all_loads` 비교 |
| 메모리 대역폭 | A vs B 의 `(L3_MISS × 64 byte) / wall_time` |
| kernel throughput | A vs B 의 tokens/s |
| numerical | atol/rtol 통과 (양 옵션 모두) |

채택 기준 — kernel throughput 가 더 높은 옵션. 단 L3 miss rate 차이가 크면 (memory-bound) memory bandwidth 적은 옵션 우선.

## 1. TL;DR

cold path 진입 시 reduced Q (D2H) 가 한 번에 transfer → 그 후 CPU kernel 시작. 본 TSK 는 reduced Q 를 N 개 chunk 로 split, chunk 별로 D2H + CPU kernel 을 stream pipeline. 첫 chunk 도착 시점부터 CPU 작업이 시작되므로 *총 wall time = max(D2H_total, CPU_total)* 로 줄어듦 (vs 현재 *D2H_total + CPU_total*).

## 4. 검증 — `TST_006`

자세한 spec 은 [`TST_006`](TST_006.md). 핵심:

- **단순 입출력 아님** — chunk N 의 D2H 가 *완료 전* 에 CPU kernel 이 chunk N 데이터로 시작함을 stream timeline 으로 측정 (CUDA event start/end vs CPU kernel start time).
- chunk 수 ∈ {1, 2, 4, 8, 16} 의 cold path 총 시간 비교 — chunk ↑ → 시간 ↓ 곡선이 plateau 되는 지점이 *최적 chunk 크기* (README §12 Q6 답).
- chunk 수 1 (현재 코드) 와 동일 결과 (numerical equivalence) — partial LSE 합산이 정확.

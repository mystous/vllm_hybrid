## 1. 검증 차원

### 1.1 · 정확도 — 두 sub-batch 결과 vs sequential reference (binding)

| 검증 항목 | 기준 |
|---|---|
| D-i (생성 token 차이) | `worst_diverging_tokens ≤ 10` |
| D-ii (logprob/PPL diff) | `worst_max_abs_logprob ≤ 0.5`, `ppl_relative_diff ≤ 0.1`, `d_ii_pass ≥ 27/30` |

→ pipelined mode 의 결과가 sequential mode (= single SubBatch GPU only) 와 *분포 동등*.

### 1.2 · 시간 매칭 — GPU prefilling vs CPU decoding attention

asymmetric pipelining 의 핵심 — *GPU prefilling 시간 ≈ CPU decoding attention 시간*:

| 시나리오 | 측정 | 목표 |
|---|---|---|
| input_heavy (1000 prompt_len, varying output_len) | `cpu_time / gpu_time` 비율 | 0.8 ~ 1.2 (= 매칭률 ≥ 80%) |
| output_heavy | 동일 | 0.8 ~ 1.2 |
| equal | 동일 | 0.8 ~ 1.2 |

매칭률 < 80% 면 PerfPredictor (TSK_017) 의 prediction 정확도 영역 또는 mode_selector 의 결정 영역에 문제.

### 1.3 · Throughput — pipelined vs sequential gain

NEO 의 정량 결과 reference:

| GPU | NEO claim | 본 TST 목표 |
|---|---|---|
| T4 (16 GB, LLaMa-2-7B) | up to 7.5× | reproduce ≥ 5× |
| A10G (24 GB, LLaMa-3-8B) | up to 26% | reproduce ≥ 15% |
| H100 (80 GB, Llama-70B + TP=8) | claim 14% — paper 수치, 본 환경 prod target | gain ≥ 8% |

본 환경 (Xeon SPR + H100×8 + Llama-70B + TP=8) 에서 throughput gain 측정.

### 1.4 · 회귀 — single sub-batch path 무회귀

scheduler (TSK_014) 가 sequential mode 를 채택한 경우 single SubBatch path 가 vanilla 와 동등 동작.

## 1. TL;DR

NEO 식 architecture 의 *CPU sub-batch attention* path 에 IDE_006 의 기존 kernel 인프라 (TSK_001/003/004/007/010) 를 통합. **NEO 의 ISPC AVX2 kernel 보다 IDE_006 의 AVX-512/AMX kernel 이 prod target (Xeon SPR) 에서 우위** 이므로 NEO 의 kernel 을 *port 하지 않음*. 단 인터페이스 변경 — IDE_006 의 *LSE-반환* 을 제거 (NEO 는 partial 합산 안 함, attention 결과 통째 반환).

### 2.3 · NEO 의 ISPC kernel 영역 — 미통합

NEO 의 `pacpu/pacpu.ispc` 는 본 TSK 의 *알고리즘 reference* 로만 사용. 코드 자체는 통합 안 함. 사유:
- NEO 의 ISPC = AVX2 (256-bit), IDE_006 의 AVX-512 (512-bit) + AMX (16 KB tile) 가 prod 우위
- 모델별 빌드 (`build.sh <model> <tp>`) 가 vLLM 의 *model-agnostic* 정책과 충돌
- IDE_006 의 dynamic dispatch (cpuid 게이트 + JIT load) 가 더 유연

## 4. 알고리즘 reference

[`NEO_code_deepdive.md`](NEO_code_deepdive.md) §7.

NEO 와 IDE_006 의 CPU kernel 비교 표:

| 차원 | NEO `pacpu` | IDE_006 (TSK_001/003) | 채택 |
|---|---|---|---|
| SIMD 폭 | AVX2 (256-bit) | AVX-512 (512-bit) + AMX 16 KB tile | **IDE_006** |
| 모델별 빌드 | 필수 | 동적 dispatch | **IDE_006** |
| GQA 처리 | inline broadcast | TSK_007 옵션 A (compact + broadcast) | **IDE_006** |
| NUMA-aware | 미적용 | TSK_004 적용 | **IDE_006** |
| Multi-OMP-team | 미적용 | TSK_010 인프라 | **IDE_006** |
| LSE 반환 | 안 함 (output 만) | 함 (LSE merge 위해) | NEO 식으로 변경 (LSE 제거) |

→ **IDE_006 의 AVX-512/AMX kernel + NUMA + sub-batching 그대로 사용**, *LSE 반환만 제거*.

핵심 변경 — `forward_attention` 의 인터페이스:

```python
# 기존 (IDE_006 TSK_001/003)
def forward_partial_with_lse(q, k_cache, v_cache, block_table, seq_lens):
    return output, lse  # LSE 반환

# 변경 (NEO 식 적용)
def forward_attention(q, k_cache, v_cache, block_table, seq_lens, output):
    # output 을 in-place 채움. LSE 미반환
    ...
```

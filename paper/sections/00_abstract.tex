\begin{abstract}
투기적 디코딩(speculative decoding)은 LLM 추론 처리량을 크게 향상시키지만,
워크로드 유형에 따라 효과가 극도로 다르게 나타난다.
특히 코드 생성 워크로드에서는 n-gram 기반 드래프터가 catastrophic regression
($-20.2\%$)을 일으키며, code agent와 같은 최신 응용에서는 이 문제가 더욱 심각하다.
suffix 기반 방법이 코드 워크로드의 한계를 완화하지만,
모델 아키텍처와 워크로드 조합에 따라 여전히 성능 손실이 발생한다.
한편 기존의 워크로드 인지 라우팅 시스템(예: llm-d)은 이 문제를 다루지만
disaggregation 구조의 복잡성이 배포 장벽으로 작용한다.

본 논문은 \algname\ (\textbf{C}ontext-aware \textbf{E}fficient \textbf{R}outing
with \textbf{E}xploited \textbf{S}lack)을 제안한다.
\algname 은 두 가지 설계 원리를 통합한다.
첫째, 워크로드 유형·모델 크기·벤더 아키텍처를 기준으로
요청별(per-request) 투기적 디코딩 방법을 자동 선택하는 \emph{워크로드 레벨 게이팅}이다.
둘째, 게이팅이 필연적으로 생성하는 이중 백엔드(dual-backend) 구조에서
유휴 CPU를 NUMA 바인딩과 작업 패턴 조정으로 활용하여
CPU-GPU 자원 간섭을 줄이고 GPU feeding 효율을 높이는 \emph{CPU 슬랙 하베스팅}이다.
이는 CPU에서 별도 추론을 수행하는 것이 아니라,
이중 백엔드의 부산물인 유휴 CPU를 간접적으로 활용하는 것이다.

H100 $\times$ 8 환경에서 Llama-3.3-70B 모델로 측정한 결과,
\algname 은 baseline 대비 sonnet 워크로드에서 $+52.1\%$,
chat에서 $+68.9\%$, code에서 $+18.8\%$의 처리량 향상을 달성하며
6개 워크로드 전체에서 순 양성(net positive)을 유지한다.
CPU 슬랙 하베스팅에서는 NUMA 바인딩이 multi-run 검증을 통과한
$+3.13\%$(warm-only)의 robust 이득을 제공하며,
스텝 경계 정렬 보조 작업은 최대 $+5.28\%$(3-mix 평균)의
예비 결과를 보인다(추가 검증 예정).
코드는 보조 자료로 공개된다.

\smallskip
\noindent\textbf{Index Terms}---
LLM inference, Speculative decoding, Workload-aware routing,
CPU-GPU heterogeneous serving, NUMA-aware optimization.
\end{abstract}

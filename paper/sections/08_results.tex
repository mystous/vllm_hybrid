\subsection{Trident Core 효과: 워크로드별 처리량}
\label{subsec:res-trident}

\input{figures/fig2_throughput_gain}

Fig.~\ref{fig:throughput-gain}과 Table~\ref{tbl:trident-vs-vanilla}는
Trident core (SuffixDecoding $+$ cudagraph PIECEWISE)의 워크로드별 처리량을 보여준다.
Llama-3.3-70B + TP=8 + H100$\times$8 환경(SUB\_093)에서
vanilla 대비 처리량 향상은 다음과 같다.

\begin{itemize}
\item \textbf{chat}: $2{,}267 \to 3{,}830$\,tps\ ($\mathbf{+68.9\%}$) ---
  $\alpha=92.7\%$, $K=10.06$으로 드래프터 효율이 최대.
\item \textbf{sonnet}: $7{,}679 \to 11{,}677$\,tps\ ($\mathbf{+52.1\%}$) ---
  $K=5.11$, 3-run 분산 $0.20\%$로 재현성 확인 (SUB\_089).
\item \textbf{code}: $6{,}718 \to 7{,}981$\,tps\ ($\mathbf{+18.8\%}$) ---
  suffix가 ngram의 catastrophic regression($-20.2\%$)을 완화.
\item \textbf{mix-bal} (34:33:33): $6{,}054 \to 9{,}514$\,tps\ ($\mathbf{+57.2\%}$) ---
  혼합 트래픽에서도 전 워크로드 양성.
\end{itemize}

wall-clock 시간은 config-wide 평균 $31\%$ 단축되었으며
($2{,}751 \to 1{,}892$\,s, SUB\_093),
GPU util은 $93.8\% \to 73.3\%$로 감소했으나
처리량은 오히려 증가했다.
이는 spec decode가 GPU step당 여러 토큰을 출력함으로써
GPU가 짧게 쉬어도 총 처리량이 늘어나는 구조를 확인한다.

\subsection{CPU 슬랙 하베스팅 결과}
\label{subsec:res-harvest}

\input{tables/tbl_harvest_results}

\textbf{NUMA 바인딩}: 3-run mean $+2.24\%$, warm-only $+3.13\%$ (SUB\_183, SUB\_194).
유일하게 multi-run binding을 통과한 lever이다.
원인은 NUMA 거리 비대칭(local 10, remote 21 = $2.1\times$)의
하드웨어 수준 일관 효과로, 측정마다 안정된 latency 절감이 발생한다.

\textbf{Branchy $\times$ 100\,ms}: 1-run 3-mix avg $+5.28\%$,
balanced cell 단독 $+12.04\%$ (SUB\_196B).
$+5\%$ 임계점에 도달한 유일한 단일 CPU lever이다.
multi-run binding 검증은 미완료이나, noise floor($\pm 3\%$) 대비 충분한
신호 크기로 유의미한 효과로 판단한다.

\textbf{Cross-domain pair (NUMA $\oplus$ softmax aux)}: $+2.83\%$,
예측값 $+3.38\%$ 대비 residual $-0.55$\,pp (SUB\_197).
code-heavy 단일 셀에서 $+7.73\%$.
교차 도메인 stacking의 near-linear superposition을 실측으로 확인한다
($\varepsilon_{\mathrm{cross}} = 0.55$\,pp $\ll$ $\varepsilon_{\mathrm{same}} = 1.18$\,pp).

\textbf{실패 패턴 요약.}
drop-in CPU kernel 7종은 microbench $1.5\text{--}5\times$ 가속에도
e2e 변환이 $0/7$로, H2D/D2H 동기화 비용($\sim 5.5$\,ms/step)이
kernel 이득을 상쇄한다.
4-OS-lever stacking은 $-0.03\%$ (residual $-1.18$\,pp),
3-lever 동일 도메인은 $-4.56$\,pp destructive이다.
task-pool phase-burst는 trident backend에서 $-14\sim-20\%$ catastrophic으로,
task-pool lock contention이 step 내부 critical path를 stall시킨다.

\subsection{Gating 이득의 원천 분해}
\label{subsec:gating-origin}

AGSD-gated가 단일 backend 대비 우위를 보이는 두 원천을 정량화한다.

\begin{enumerate}
\item \textbf{병렬 GPU 활용 ($\sim 50$--$60\%$ 기여)}:
  두 backend가 동시에 활성화되어 이론상 $2\times$ GPU 자원이 서빙에 투입된다.
  Qwen-32B balanced에서 vanilla-only $1{,}779$ tps $\to$ trident-only $2{,}721$ tps의
  $+53\%$가 이미 backend 병렬화만으로 발생한다.

\item \textbf{Gating decision ($\sim 10$--$20\%$ 기여)}:
  per-request 방법 선택이 회귀 방지(소형 모델 chat $\to$ vanilla)와
  최대 가속 유지(code/sonnet $\to$ Trident)를 동시에 달성한다.
  Qwen-7B에서 gating value가 최대($+25.3\%$ vs.\ trident-only)인 이유는
  chat 워크로드의 vanilla 선택 효과가 특히 크기 때문이다.
\end{enumerate}

\subsection{소형 모델 회귀 하드웨어 독립성 검증}
\label{subsec:res-hw-independent}

vLLM Issue \#16258은 opt-125m + ngram이 L4$\times$2에서 $2.12\times$ wall
regression을 보임을 보고하였다.
본 연구는 H100$\times$1 환경에서 동일 모델로 $2.13\times$ wall을 측정하여
정확히 재현하였으며 (SUB\_091), 추가로 4개 모델을 cross-validation하였다.

회귀율은 starcoder2-3b $2.30\times$, Qwen-0.5B $2.46\times$,
Qwen-1.5B $2.63\times$로 증가하며, Qwen-0.5B + suffix도 $2.06\times$를 기록한다.
이 패턴은 H100과 L4 두 GPU 세대에서 일관되므로,
소형 모델 투기적 디코딩 회귀가 GPU 아키텍처에 무관한 \emph{구조적 현상}임을
5개 모델 cross-validation으로 확립한다.
이는 \algname 의 모델 크기 게이팅($\leq$7B $\to$ vanilla)의 정량적 근거이다.

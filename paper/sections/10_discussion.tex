\subsection{llm-d와의 비교}

워크로드 인지 라우팅을 지원하는 강력한 접근이다.
두 시스템의 설계 철학은 다음과 같이 대비된다.

\subsection{한계}

\textbf{하드웨어 특화.}
본 측정은 H100$\times$8 + Xeon SPR 듀얼-NUMA 환경에 특화되어 있다.
다른 GPU 세대(A100, H200), AMD GPU, 단일 소켓 서버에서
하베스팅 hyperparameter($T_{\text{aux}}$, branchy 패턴)를 재조정해야 할 수 있다.
\algname 의 $T_{\text{gpu}}$ 자동 측정이 부분적 일반화를 지원하지만
multi-environment 검증은 후속 연구이다.

\textbf{Multi-run binding 미완료.}
Branchy $\times$ 100ms의 $+5.28\%$(1-run)가 multi-run에서도 유지되는지
SUB\_194 유형의 추가 측정이 필요하다.
현재 magnitude($+5.28\%$, noise floor $\pm 3\%$ 초과)는 의미 있는 신호로 판단하나
production 배포 전 검증을 권장한다.

\textbf{Gating 규칙의 확장성.}
현재 벤더-워크로드 게이팅 규칙(Qwen-72B code $\to$ vanilla)은
수동 측정 결과에서 도출된다.
Llama-405B FP8, 새 아키텍처 등 미측정 모델에서
runtime accept rate 모니터링 기반 자동 규칙 생성이 필요하다.

\subsection{LLM 추론 서빙의 자원 비대칭 문제}

최근 대규모 언어모델(LLM) 의 상용 서빙 시스템은 PagedAttention 기반의 vLLM~\cite{kwon2023vllm} 과 Continuous Batching~\cite{yu2022orca} 의 보편화에 의해 GPU 활용률 측면에서 큰 진전을 이루었다.
나아가 투기적 디코딩(speculative decoding)~\cite{leviathan2023spec, chen2023spec, stern2018blockwise} 이 대중적으로 도입되면서, 동일한 GPU 자원으로 처리되는 토큰 수가 모델 크기와 워크로드에 따라 2--3배까지 증가하는 결과가 보고되고 있다~\cite{cai2024medusa, li2024eagle}.

그러나 GPU 자원의 효율적 활용과 대조적으로, GPU 가속 노드에 함께 배치된 호스트 CPU 자원은 평균 4--5\,\% 수준에서 머무는 비대칭이 측정 자료를 통해 일관되게 관찰된다.
이는 GPU 가속 LLM 추론의 critical path 가 GPU forward와 attention 연산이 dominant 이며, 호스트 CPU 는 토크나이저(tokenizer), 결과 detokenization, 워크로드 라우팅 등 비교적 가벼운 부차 작업에만 사용되기 때문이다.
이러한 비대칭 환경에서 \emph{어떻게 CPU 자원을 활용해 시스템 전체의 처리량을 추가로 회수할 것인가} 는 LLM 추론 서빙 분야의 중요한 미해결 문제로 남아 있다.

\subsection{24-기법 체계적 탐색의 결과}

본 연구는 위 문제에 대한 답을 찾기 위해 CPU 자원을 활용하는 24가지 후보 최적화 기법을 5개 분류군---(A) 직접 CPU 커널 대체, (B) 시스템 수준 격리, (C) sub-step CPU 스케줄링, (D) 보조 워크로드 오프로딩, (E) 동시 보조 작업---으로 체계화하고 paired control/treatment 측정을 통해 각 기법의 단독 효과를 분석하였다.

측정 결과 다음 세 가지 음성 현상이 일관되게 관찰되었다.
첫째, 24개 단일 기법 중 어떤 기법도 처리량 +5\,\% 의 유의미한 임계점에 도달하지 못했다.
둘째, microbench 수준에서 1.5--5배의 속도 향상을 보였던 직접 CPU 커널 대체 기법들이 end-to-end 처리량으로 변환되는 비율은 0/7 로 일관되게 실패하였다.
셋째, 단순한 기법 조합 stacking 은 동일 도메인일 경우 destructive interference 를 발생시켜 단독 기법 합산보다 더 낮은 효과를 보였다.

\subsection{본 논문의 기여}

본 논문은 위 음성 결과들을 단순 보고에 그치지 않고, 그 안에서 발견된 두 가지 비자명한 현상을 출발점으로 통합 알고리즘 \algname 을 제안한다.

\textbf{Contribution 1: \algname 알고리즘.}
GPU step 주기에 정렬된 CPU 자원 오케스트레이션 알고리즘을 음악적 metaphor 로 형식화한다.
5단계 파이프라인---\emph{Tempo} (박자 측정), \emph{Chord} (교차 도메인 기법 쌍 선택), \emph{Meter} (CPU 주기 정렬), \emph{Accent} (작업 패턴 선택), \emph{Ritardando} (drift 시 재조정)---은 24개 단일 기법이 모두 미달한 5\,\% 임계점을 단일 구성에서 도달한다.

\textbf{Contribution 2: 교차 도메인 중첩 정리.}
LLM 추론 서빙에서 CPU 최적화 기법의 stacking 효과를 처음으로 정량적으로 정식화한다.
도메인 간섭 비용 $\varepsilon(A,B)$ 은 동일 도메인 stacking 시 약 1.18 pp, 교차 도메인 시 0.55 pp, 동일 도메인 3+ stack 시 4.56 pp 의 초선형 destructive 패턴을 나타냄을 경험적으로 보였다.

\textbf{Contribution 3: 분기 패턴 + 저빈도 cycle 의 역설.}
직관과 반대로, 고빈도 regular work 보다 저빈도 branchy work 가 5.4배 높은 효과를 나타내는 현상을 발견하고 CPU prefetcher 의 GPU PCIe 트래픽과의 contention 회피 메커니즘으로 설명한다.
